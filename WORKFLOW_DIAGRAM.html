MedSentinel Workflow Diagram

Complete User Journey & System Architecture

User Journey Flow

1

Landing Page (index.html)

User visits homepage → Views features, schemes, hospitals → Clicks "Compare" button

2

Search Flow (search-flow.html)

Step 1: Medical Condition Search (condition, location, urgency)

Step 2: Patient Profile Details (name, age, gender, income, state, aadhaar)

Step 3: AI Processing Animation (10-15 seconds)

3

Hospital Search Results (hospitals.html)

View filtered hospitals → Apply filters (distance, rating, schemes) → Save hospitals → View details

4

Hospital Details (hospital-details.html)

View cost breakdown → Check facilities → See accepted schemes → Book consultation or Apply for scheme

5A

Book Consultation

Fill booking form → Select date/time → Choose department → Submit → Confirmation

5B

Government Schemes

Browse schemes → Apply filters → Check eligibility → View application process

6

Eligibility Check (eligibility.html)

Enter household details → Select family size, location, income → AI analyzes eligibility → View eligible schemes

7

Cost Comparison (comparison.html)

Compare costs across hospitals → View cost breakdown → Analyze savings → Select best option

8

User Profile (profile.html)

View saved hospitals → Manage schemes → Check search history → Upload documents

Backend Architecture

Server (server.js)

  • • Express.js on Port 3001
  • • SQLite Database
  • • CORS enabled
  • • Body parser middleware

Database Tables

  • • hospitals (10 fields)
  • • users (7 fields)
  • • schemes (5 fields)
  • • saved_items (3 fields)

API Endpoints

GET /api/hospitals

GET /api/hospitals/:id

GET /api/schemes

POST /api/users

GET /api/users/:id

POST /api/saved-items

GET /api/saved-items/:userId

DELETE /api/saved-items/:userId/:hospitalId

POST /api/eligibility-check
